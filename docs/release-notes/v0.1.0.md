# DropRaw Web v0.1.0 — Multi-cloud RAW to Web Assets

## Highlights

- Convert large RAW folders from Dropbox, Google Drive, S3/R2, and local folders.
- Produce WebP, AVIF, JPEG, and PNG assets.
- Resume safely after network failures.
- Avoid huge folder ZIP downloads.

## Why This Exists

This release started from a real Dropbox shared folder with 816 Nikon NEF files. Transient listing and download failures made retry/resume behavior a core requirement, not a cosmetic feature.

## Supported Sources

Local folders, Dropbox shared folders, Google Drive folders, AWS S3, Cloudflare R2, and S3-compatible storage.

## Supported Output Formats

WebP, AVIF, JPEG, and PNG. AVIF depends on Pillow build support.

## Dropbox Reliability Work

DropRaw Web handles listing retries, download retries, exponential backoff, `.part` downloads, size checks, failed logs, retry only failed, and resume without overwriting existing outputs.

Recoverable transient errors such as `ConnectionError(ProtocolError('Connection aborted.', OSError(22, 'Invalid argument')))` are retried.

## Local UI

`dropraw ui` starts a local dashboard for source setup, scan, conversion settings, retry settings, progress, failed files, and reports.

## Reports

Reports include JSON, CSV, HTML, assets JSON, errors CSV, failed TSV, and HTML picture snippets.

## International Mirrors

GitHub is primary. GitLab, Codeberg, and Bitbucket are documented as international mirrors.

## Russian Mirrors

GitFlic and GitVerse are documented as Russian mirrors. Сфера.Код is documented as an enterprise/self-hosted target. РТК-Феникс is documented as a secure supply-chain/dependency context, not as a fake Git mirror.

## Installation

```bash
pip install dropraw-web
```

Development:

```bash
git clone https://github.com/hemp-dev/dropraw-web.git
cd dropraw-web
python3.12 -m venv .venv
source .venv/bin/activate
pip install -e ".[dev]"
```

## Known Limitations

- OAuth flows may still require manual setup depending on provider.
- AVIF depends on Pillow build support.
- Python 3.14 is not recommended yet.
- OneDrive, Yandex Disk, and Box are roadmap or experimental unless explicitly verified.
- Some mirrors are documented as optional until verified.
- Russian enterprise platforms may require private/internal setup.

## What's Next

OAuth improvements, more providers, richer UI queue management, better metadata controls, and verified mirrors across selected platforms.
